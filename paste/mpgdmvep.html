上传数据

题目背景

题目来自图灵杯，数据来自 LZJ。

题目描述

出题屡屡令人不爽，尤其是上传数据传不上、不让传时。

小 CBC 出了一道题，试图上传数据，然而不幸的是数据可能会爆上传空间限制，于是就先挑了几个必须上传的测试点。

简单来说，数据分为 nnn 个 Subtask，第 iii 个 Subtask 共造了 pip_ipi​ 个测试点。为了保证数据强度，必须上传至少 wiw_iwi​ 个测试点。然而，由于上传数据的总空间限制，我们总计上传的测试点数不能超过 mmm 个。

换句话说，假设第 iii 个 Subtask 上传了 qiq_iqi​ 个数据，那么 qiq_iqi​ 至少为 wiw_iwi​，最多为 pip_ipi​，并且 qiq_iqi​ 的总和不能超过 mmm。

在这基础上，小 CBC 希望最小的一个 qi/piq_i/p_iqi​/pi​ 尽可能大。你可以帮帮他吗？

输入格式

第一行两个整数 n,mn,mn,m。

接下来 nnn 行，每行两个整数 pi,wip_i,w_ipi​,wi​。

输出格式

若无方案满足，请输出 Zip file size too big after extraction.Poor CBC!。

否则第一行以最简分数形式输出最大化的 qipi\dfrac{q_i}{p_i}pi​qi​​ 最小值（形如 x/y，使得 gcd⁡{x,y}=1\gcd\{x,y\}=1gcd{x,y}=1；特别地，111 为 1/1，000 为 0/1）。第二行输出你构造的数列 q1∼qnq_1\sim q_nq1​∼qn​ 。

注意第一行行末不可有多余空格。
