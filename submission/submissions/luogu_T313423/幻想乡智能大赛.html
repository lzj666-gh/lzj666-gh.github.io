幻想乡智能大赛

题目描述

命莲寺的众人为了吸引参拜客来参拜，造了一个 n×mn\times mn×m 的迷宫，迷宫外围是一层墙壁。第一个到达终点的人，可以获得宝贝。

于是幻想乡里的妖怪神仙全来这里大显神通。不过对门的神灵庙一众不乐意了：生意都被抢走了！

所以神灵庙打算来砸场子。具体方式是首先到达终点抢到宝贝。

迷宫起点在 (1,1)(1,1)(1,1) 位置，需要到达 (n,m)(n,m)(n,m) 位置，每一回合可以向前后左右四个位置移动。

然而命莲寺众人不会轻易地让选手获得奖品，所以设下了结界，在任何位置只能看到前后左右四个方向有没有墙。对于这道题而言，你只能通过交互器来判定四周的情况。在刚进入迷宫时，以及每一步行动后，交互器将会告诉你四周墙壁的情况。格式见「交互细节」处。

现在需要花尽可能少的步数到达终点。如果不可能到达终点，请输出 Baka\texttt{Baka}Baka。

这和人工智能有什么关系？

专业小偷霍青娥从命莲寺偷到了迷宫构造器，并且造了一个有 100001000010000 组数据的大样例，实际迷宫使用相同的构造器制造。可以使用这些数据训练你的程序。

交互细节

对于每一组数据，在你第一次行动前，以及每次行动后，你将得到一些输出提示你周围墙壁的情况：

  • 如果你得到的输出是 −1-1−1，那么说明你进入了墙内。请立即终止程序，以防出现与预期相悖的事情。
  • 否则你将得到四个整数 x1,x2,x3,x4x_1,x_2,x_3,x_4x1​,x2​,x3​,x4​。
    • 如果 x1=1x_1=1x1​=1，那就说明正上方有一堵墙壁，否则值为 000 表示没有墙壁；
    • 如果 x2=1x_2=1x2​=1，那就说明正右方有一堵墙壁，否则值为 000 表示没有墙壁；
    • 如果 x3=1x_3=1x3​=1，那就说明正下方有一堵墙壁，否则值为 000 表示没有墙壁；
    • 如果 x4=1x_4=1x4​=1，那就说明正左方有一堵墙壁，否则值为 000 表示没有墙壁。
  • 当你到达 (n,m)(n,m)(n,m) 位置后，此轮胜利。交互库会立即终止，你的程序也应该立即退出。同时，你不应该输出任何多余的东西。

输入格式

这里描述的是下发的 maze.data 的格式。

第一行有一个正整数 TTT，表示数据组数。对于每组数据，

  • 第一行有两个整数 n,mn,mn,m，描述迷宫的大小。
  • 接下来 nnn 行，每行有 mmm 个字符 #\verb!#!# 或者 .\verb!.!.。如果为 #\verb!#!# 则表示此处是一个墙壁，否则是空气。

输出格式

每一次行动时，你可以输出以下 444 个字符串之一来移动：

  • North\mathtt{North}North：向正上方移动一格；
  • South\mathtt{South}South：向正下方移动一格；
  • West\mathtt{West}West：向正左方移动一格；
  • East\mathtt{East}East：向正右方移动一格。

特别地，如果你根据已有信息断定迷宫一定没有路径，请输出 Baka\verb!Baka!Baka。

判分方式

记从 (1,1)(1,1)(1,1) 到达 (n,m)(n,m)(n,m) 的最短路径长度为 d0d_0d0​。你走过的路径长度为 d1d_1d1​。

  • 若 ∣d1−d0∣≤6|d_1-d_0|\le 6∣d1​−d0​∣≤6，你将获得该测试点全部分数；
  • 若 ∣d1−d0∣≤103|d_1-d_0|\le 10^3∣d1​−d0​∣≤103，你将获得该测试点 50%50\%50% 分数；
  • 若 ∣d1−d0∣≤106|d_1-d_0|\le 10^6∣d1​−d0​∣≤106，你将获得该测试点 10%10\%10% 分数；
  • 否则，你将不能获得任何分数。

数据范围及约定

  • 对于 20%20\%20% 的数据，1≤n,m≤31\le n,m\le 31≤n,m≤3；
  • 对于 40%40\%40% 的数据，1≤n,m≤51\le n,m\le 51≤n,m≤5；
  • 对于 80%80\%80% 的数据，1≤n,m≤241\le n,m\le 241≤n,m≤24；
  • 对于全部数据，1≤n,m≤1281\le n,m\le 1281≤n,m≤128。
